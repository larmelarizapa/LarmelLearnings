What are Annotations?

Description: Annotations in Java are metadata that provide information about the code but do not change its behavior. They are often used by frameworks and tools during compilation, deployment, or runtime.


  // Example of annotation usage
  @Override
  public String toString() {
    return "Hello!";
  }

Common Built-in Annotations

Description: Java provides several built-in annotations:

  • @Override – Indicates a method overrides a superclass method.
  • @Deprecated – Marks a method/class as outdated.
  • @SuppressWarnings – Tells the compiler to ignore warnings.

  class MyClass {
    @Deprecated
    void oldMethod() {
      System.out.println("This method is deprecated.");
    }
  
    @SuppressWarnings("unchecked")
    void useList() {
      List rawList = new ArrayList();
    }
  }

Creating Custom Annotations

Description: You can create your own annotations using the @interface keyword.


  import java.lang.annotation.*;
  
  @Retention(RetentionPolicy.RUNTIME)
  @Target(ElementType.METHOD)
  public @interface MyAnnotation {
    String value();
  }

Using Custom Annotations

Description: Apply custom annotations and access them using reflection.


  public class Demo {
    @MyAnnotation(value = "Hello Larmel!")
    public void show() {
      System.out.println("Method with annotation.");
    }
  }

Accessing Annotations with Reflection

Description: Use Java reflection to read annotation values at runtime.


  import java.lang.reflect.*;
  
  public class Main {
    public static void main(String[] args) throws Exception {
      Method method = Demo.class.getMethod("show");
      MyAnnotation annotation = method.getAnnotation(MyAnnotation.class);
  
      System.out.println(annotation.value()); // Output: Hello Larmel!
    }
  }
