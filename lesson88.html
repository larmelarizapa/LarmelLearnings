Introduction to Modules

Description: JavaScript modules allow you to split your code into smaller, reusable pieces. This helps improve code organization, readability, and maintainability by isolating functionality into separate files.

Importing and Exporting Modules

Description: Modules are typically divided into two types of operations: export (to make a piece of code available for use in other modules) and import (to bring code from another module into the current file).


  // Exporting from module1.js
  export const greet = (name) => `Hello, ${name}!`;
  
  // Importing into main.js
  import { greet } from './module1.js';
  console.log(greet("Larmel")); // "Hello, Larmel!"

Default Exports

Description: A module can export a single entity using a default export. This allows you to import the module without specifying a named reference.


  // Exporting a default function in module1.js
  export default function greet(name) {
    return `Hello, ${name}!`;
  }
  
  // Importing default export in main.js
  import greet from './module1.js';
  console.log(greet("Larmel")); // "Hello, Larmel!"

Exporting Multiple Elements

Description: You can export multiple functions, objects, or variables from a module using named exports. This allows the importing module to choose which specific pieces of code to bring in.


  // module1.js
  export const add = (a, b) => a + b;
  export const subtract = (a, b) => a - b;
  
  // main.js
  import { add, subtract } from './module1.js';
  console.log(add(5, 3));     // 8
  console.log(subtract(5, 3)); // 2

Renaming Imports

Description: You can rename imported functions or variables to avoid name conflicts or provide better clarity.


  // module1.js
  export const greet = (name) => `Hello, ${name}!`;
  
  // main.js
  import { greet as sayHello } from './module1.js';
  console.log(sayHello("Larmel")); // "Hello, Larmel!"

Importing All Exports

Description: You can import all the exported values from a module into an object using the * as syntax.


  // module1.js
  export const add = (a, b) => a + b;
  export const subtract = (a, b) => a - b;
  
  // main.js
  import * as math from './module1.js';
  console.log(math.add(5, 3));     // 8
  console.log(math.subtract(5, 3)); // 2

Dynamic Imports

Description: Dynamic imports allow you to load a module at runtime using the import() function. This helps optimize performance by loading modules only when needed.


  // Dynamic import example
  function loadGreetModule() {
    import('./module1.js').then((module) => {
      console.log(module.greet("Larmel"));
    });
  }
  
  loadGreetModule(); // "Hello, Larmel!"

Using Modules in HTML

Description: To use modules in HTML files, include the type="module" attribute in your script tag. This enables ES6 module functionality in the browser.

Practical Example of Using Modules

Description: Below is an example of how modules can be used in a project with multiple files to organize functionality like calculations and greeting messages.


  // module1.js
  export const greet = (name) => `Hello, ${name}!`;
  
  // module2.js
  export const add = (a, b) => a + b;
  export const subtract = (a, b) => a - b;
  
  // main.js
  import { greet } from './module1.js';
  import { add, subtract } from './module2.js';
  
  console.log(greet("Larmel"));
  console.log(add(5, 3));     // 8
  console.log(subtract(5, 3)); // 2
