What are Promises?

Description: A Promise is an object that represents the eventual completion (or failure) of an asynchronous operation and its resulting value. It can be in one of three states: pending, resolved (fulfilled), or rejected.


  // Creating a simple Promise
  let promise = new Promise(function(resolve, reject) {
    let success = true;
  
    if (success) {
      resolve("The operation was successful!");
    } else {
      reject("The operation failed.");
    }
  });
  
  promise.then(function(result) {
    console.log(result); // "The operation was successful!"
  }).catch(function(error) {
    console.log(error);  // "The operation failed."
  });

Understanding Promise States

Description: A Promise can be in one of three states:

  • Pending: Initial state, neither fulfilled nor rejected.
  • Fulfilled: The operation was successful, and the promise is resolved.
  • Rejected: The operation failed, and the promise is rejected.

  // Example showing pending, fulfilled, and rejected states
  let fetchData = new Promise(function(resolve, reject) {
    let dataFetched = true;
  
    if (dataFetched) {
      resolve("Data fetched successfully!");
    } else {
      reject("Failed to fetch data.");
    }
  });
  
  fetchData
    .then(function(result) {
      console.log(result);  // "Data fetched successfully!"
    })
    .catch(function(error) {
      console.log(error);   // "Failed to fetch data."
    });

Chaining Promises

Description: You can chain multiple then() methods to handle successive asynchronous operations. Each then() returns a new promise.


  let promise = new Promise(function(resolve, reject) {
    resolve(5);
  });
  
  promise
    .then(function(result) {
      console.log(result); // 5
      return result * 2;
    })
    .then(function(result) {
      console.log(result); // 10
      return result + 3;
    })
    .then(function(result) {
      console.log(result); // 13
    });

Async Functions and Await

Description: async is used to declare a function that will return a Promise, and await pauses the execution of the function until the Promise is resolved or rejected. This makes asynchronous code look more like synchronous code.


  async function fetchData() {
    let data = await new Promise(function(resolve, reject) {
      let success = true;
      setTimeout(function() {
        if (success) {
          resolve("Data loaded successfully!");
        } else {
          reject("Failed to load data.");
        }
      }, 2000);
    });
  
    console.log(data); // "Data loaded successfully!"
  }
  
  fetchData();

Handling Errors with Async/Await

Description: You can handle errors in asynchronous code using try...catch blocks when working with async functions. This is much cleaner and more readable than using then() and catch().


  async function fetchData() {
    try {
      let data = await new Promise(function(resolve, reject) {
        let success = false;
        setTimeout(function() {
          if (success) {
            resolve("Data loaded successfully!");
          } else {
            reject("Failed to load data.");
          }
        }, 2000);
      });
  
      console.log(data);
    } catch (error) {
      console.log(error); // "Failed to load data."
    }
  }
  
  fetchData();

Using Async/Await with Multiple Promises

Description: You can use async/await with multiple promises by using Promise.all() to execute multiple asynchronous operations concurrently.


  async function fetchMultipleData() {
    try {
      let data = await Promise.all([
        new Promise(function(resolve, reject) {
          setTimeout(() => resolve("First promise resolved"), 1000);
        }),
        new Promise(function(resolve, reject) {
          setTimeout(() => resolve("Second promise resolved"), 2000);
        })
      ]);
      
      console.log(data); // ["First promise resolved", "Second promise resolved"]
    } catch (error) {
      console.log(error);
    }
  }
  
  fetchMultipleData();
