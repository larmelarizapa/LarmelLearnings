What is Multithreading?

Description: Multithreading is a Java feature that allows the concurrent execution of two or more threads. A thread is a lightweight subprocess, and multithreading is used to perform multiple tasks simultaneously for better performance.

Creating a Thread (Extending Thread Class)

Description: One way to create a thread is by extending the Thread class and overriding the run() method.


  class MyThread extends Thread {
    public void run() {
      System.out.println("Thread is running...");
    }
  
    public static void main(String[] args) {
      MyThread t1 = new MyThread();
      t1.start();
    }
  }

Creating a Thread (Implementing Runnable Interface)

Description: Another way is to implement the Runnable interface and pass the object to a Thread constructor.


  class MyRunnable implements Runnable {
    public void run() {
      System.out.println("Thread is running using Runnable...");
    }
  
    public static void main(String[] args) {
      MyRunnable myRunnable = new MyRunnable();
      Thread t1 = new Thread(myRunnable);
      t1.start();
    }
  }

Thread Lifecycle

Description: A thread has several states: New, Runnable, Running, Blocked/Waiting, and Terminated. The JVM and OS manage these states automatically based on scheduling.

  • New: Thread is created but not started.
  • Runnable: Thread is ready to run.
  • Running: Thread is executing.
  • Waiting/Blocked: Thread is paused and waiting.
  • Terminated: Thread has completed execution.

Thread Sleep

Description: The sleep() method is used to pause the execution of a thread for a specified time.


  class MyThread extends Thread {
    public void run() {
      for (int i = 1; i <= 5; i++) {
        System.out.println(i);
        try {
          Thread.sleep(1000); // Pause for 1 second
        } catch (InterruptedException e) {
          System.out.println(e);
        }
      }
    }
  
    public static void main(String[] args) {
      MyThread t1 = new MyThread();
      t1.start();
    }
  }

Thread Synchronization

Description: When multiple threads access the same resource, synchronization ensures that only one thread accesses it at a time to avoid conflicts.


  class Table {
    synchronized void printTable(int n) {
      for (int i = 1; i <= 5; i++) {
        System.out.println(n * i);
        try {
          Thread.sleep(400);
        } catch (Exception e) {
          System.out.println(e);
        }
      }
    }
  }
  
  class MyThread1 extends Thread {
    Table t;
    MyThread1(Table t) {
      this.t = t;
    }
    public void run() {
      t.printTable(5);
    }
  }
  
  class MyThread2 extends Thread {
    Table t;
    MyThread2(Table t) {
      this.t = t;
    }
    public void run() {
      t.printTable(100);
    }
  }
  
  public class TestSynchronization {
    public static void main(String args[]) {
      Table obj = new Table();
      MyThread1 t1 = new MyThread1(obj);
      MyThread2 t2 = new MyThread2(obj);
      t1.start();
      t2.start();
    }
  }
